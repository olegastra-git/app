В данной Use Case-диаграмме [User] представляет обычных пользователей мессенджера, а [Admin] – администраторов, обладающих более расширенными правами.

Варианты использования (Use Cases) для [User] остаются прежними и включают регистрацию, авторизацию, отправку и получение сообщений, управление групповыми чатами, редактирование профиля и другие действия, связанные с общением в мессенджере.

Для [Admin] добавлены следующие Use Cases:

- Создание учётной записи пользователя (Create User Account)
- Удаление учётной записи пользователя (Delete User Account)
- Обновление учётной записи пользователя (Update User Account)
- Просмотр статистики пользователя (View User Statistics)
- Управление учётными записями пользователей (Manage User Account) – этот Use Case расширяет [Admin]
- Управление групповыми чатами (Manage Group Chat)
- Блокировка пользователя (Block User)
- Обработка жалоб на пользователей (Handle User Reports) – этот Use Case расширяет [Admin]
- Просмотр отчетов (View Reports)

Между Use Cases используются отношения include и extend, чтобы указать включение и расширение определенных действий в рамках других Use Cases в соответствии с требованиями системы и функциональностью мессенджера.
